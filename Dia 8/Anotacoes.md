# Programação Orgânica VS Impressora 3D
Durante o desenvolvimento de software, é normal que um dos modelos seja escolhido, não há um melhor, mas há um certo para cada tipo de projeto.

- **Modelo Orgânico:**
    - Usado para um projeto "novo" (Algum que esteja sendo novidade agora).
    - Gera a sensação de estar contruíndo desde o zero.
    - Diminui a incidência de `over engineering` (Complicar algo que poderia ter sido programado de uma forma mais simples).
    - Evita `feature creep` (É quando um projeto possui tantos recursos ao ponto de atraplhar mais do que ajudar).
    - Funcionalidades são entregues e aprimoradas uma a uma, aos poucos.
    <br>

- **Modelo Impressão 3D:**
    - Usado para projetos que já fizemos antes (Podemos considerar um projeto que consiste em ser quase igual a um que já fizemos antes).
    - Entrega o projeto com todas `features` (Recursos) prontos.
    - Modelo de desenvolvimento mais estressante.
    <br>

**Notas:**
- Errar na escolha do modelo **pode ser fatal para o projeto**.
- Escolher o modelo é a primeira escolha do desenvolvimento.
- Oproejto do curso será cosntruido com o `modelo Orgânico`.
