## Diferença entre Git e GitHub
**Git:**
Git é uma tecnologia capaz de trabalhar com o versionamento de código, podendo rastrear alterações, criar <i>Branches</i>, <i>Forks</i>, etc..
<br>

**Nota:** `Git` não foi o primeiro sistema de versionamento a criar o sistema de `Branches`, porém, se destacou pela sua otimização.
<br>

**GitHub:**
Github é uma plataforma de armazenamento de repositórios. Cada repositório armazena um projeto de desenvolvimento.

---

## Criando um Repositório no GitHub
- Usar nome curto e memorável.
- <i>`Slug`</i> - Termo único utilizado nas URL para rotas no site.
O GitHub junta `nome do usuário` + `nome do respositório` para criação do <i>**slug**</i>.

### <i>SEO</i> **- Search Engine Optimization/Otimização para Mecanismos de Busca**
É o conjunto de técnicas usadas para fazer um site, página ou conteúdo aparecer melhor posicionado nos resultados do Google, Bing, etc., sem pagar anúncios.
<br>

**Nota:** O conceito de <i>`SEO`</i> deve aplicado ao **nome do repositório do projeto**.

## WSL - Windows Subsystem for Linux/Windows Subsistema para Linux
É um recurso do Windows que permite rodar um ambiente Linux real dentro do Windows, sem máquina virtual pesada e sem dual boot.

---

## GitHub - Codespaces
Tecnologia usada para executar um ambiente de desenvolvimento na nuvem com integração nativa de um `Visual Studio Code` e um `terminal (Linux)`.
